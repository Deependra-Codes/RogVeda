# Docs Index

This directory is split into product context and the engineering OS.

## Reading Order

1. `context/current-state.md`
2. `product/trial-brief.md`
3. `product/build-tracker.md`
4. `product/ui-bible.md` for visual and interaction work
5. `process/README.md`
6. the relevant records in `process/records/`
7. the relevant standards in `process/standards/`

## Layout

- `context/`
  fastest current-state handoff for any contributor or AI
- `product/`
  Rogveda brief interpretation, delivery tracking, and product-level context, including the canonical UI bible
- `process/`
  Standards, templates, records, and enforcement-facing documentation

## Working Rule

For non-trivial work, the repo follows:

`Intent -> Research -> Spec -> Types/Data Model -> Tests -> Implementation -> Verification`
